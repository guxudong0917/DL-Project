# 配环境

配环境自古以来就是一大难题，在昨天做学校的一个nemu实验时，因为docker环境和vscode的某种差异导致我被硬控了九个小时后，我就更坚信了这一点，所以，我决定把关于在配环境上踩到的坑也加以记录。

## 或许你需要….Anaconda，有更好的吗?

Anaconda真的有一大堆槽点，两个库总是会莫名奇妙的冲突，dll会缺失巴拉巴拉，但它确实是我们需要掌握的。

## 常见命令

创建环境

```
conda create -n 环境名 [后面可以跟着指定在环境中要配什么东西]
```

这个过程可能要很久

激活环境

```
conda activate env_name
```

退出环境

```
conda deactivate
```

显示环境

```
conda env list
```

显示环境中的所有库

```
conda list
```

下载库

```
conda install pytorch
```

删除库

```
conda uninstall pytorch
```

## what can I say？mamba ！

用mamba 来下载库会快一点，实践已经向我证明了这一点

```
mamba install pytorch
```

## 下载哪些

至少要有pytroch，这一步会把连带着需要的numpy啥的都下载

```
        mamba install pytorch torchvision cpuonly
```

jupyter notebook

```
        mamba install jupyter
```
